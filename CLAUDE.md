## 행동 규칙 (반드시 준수)

1. **설명 금지**: 사용자가 설명을 요청하기 전까지 무엇을 했는지, 왜 했는지 설명하지 않는다. 코드/결과만 출력한다.
2. **수정 전 검증 필수**: 파일을 수정하기 전에 관련 파일을 모두 읽고, 전체 실행 플로우를 추적하고, 기존 버그를 먼저 파악한다. 버그가 있으면 선처리 후 요청된 변경을 마지막에 적용한다.
3. **의도 파악 우선**: 요청이 모호하면 행동하기 전에 핵심 질문 하나만 한다. 임의로 가정하지 않는다.
4. **UI/UX 플로우 확인**: 프론트엔드 수정 시 전체 사용자 인터랙션 플로우(페이지 로드 → 지갑 연결 → 액션 → 피드백)를 먼저 추적한다. 변경사항이 인접 플로우를 깨지 않는지 확인한다.
5. **개선안 보고**: 작업 전 검증 단계에서 더 나은 접근법이나 비명백한 문제를 발견하면 작업 시작 전에 보고한다. (항목당 한 줄, 요청 전까진 상세 설명 금지)
6. **재확인 금지**: "이렇게 할까요?" 식의 확인 질문 없이 바로 실행한다.
7. **완료 메시지 금지**: "완료했습니다", "수정했습니다" 같은 완료 알림을 출력하지 않는다.
8. **Tool 호출 전 발화 금지**: "파일을 읽겠습니다" 같은 예고 문장 없이 바로 tool을 호출한다.
9. **오류 시 1회 진단**: 같은 접근법을 반복 재시도하지 않는다. 원인을 파악한 후 방향을 전환한다.

## Project Overview

Decentralized secondhand marketplace + auction platform targeting Sepolia testnet. Monorepo with two workspaces: `contracts/` (Hardhat/Solidity) and `frontend/` (Next.js/TypeScript).

## Commands

### Contracts (`cd contracts`)
```bash
npm run compile       # Compile Solidity (solc 0.8.24)
npm run test          # Run all Hardhat tests
npx hardhat test test/DirectSale.test.js   # Run a single test file
npm run export-abi    # Copy ABIs → frontend/src/abi/
npm run verify        # Verify deployed contracts on Etherscan
```

### Frontend (`cd frontend`)
```bash
npm run dev           # Start Next.js dev server
npm run build         # Production build
npm run lint          # ESLint
```

## Architecture

### Smart Contracts

**Deployment flow:** `Marketplace.sol` is the central registry. Each listing deploys a standalone trading contract (`DirectSale`, `OpenAuction`, or `BlindAuction`) and registers its address via `listItem()`.

```
Marketplace.sol          ← registry; stores listing metadata + tradeContract address
├── DirectSale.sol       ← fixed-price escrow (OnSale → Locked → Completed/Cancelled)
├── OpenAuction.sol      ← English auction; previous bidders refunded immediately
└── BlindAuction.sol     ← Vickrey (second-price sealed-bid); commit→reveal phases
InterestCalculator.sol   ← 5% annual simple interest; charged to platform on locked funds
interfaces/              ← IMarketplace, IAuction, IInterestCalculator
```

**BlindAuction commit format:** `keccak256(abi.encodePacked(bidAmount, secret))` — winner pays second-highest price, not their own bid.

All fund-transfer functions use `ReentrancyGuard`. Images are stored as IPFS hashes (`imageHash`).

### Frontend

**Web3 stack:** wagmi v2 + viem v2 + RainbowKit (WalletConnect). Contract ABIs live in `src/abi/` (generated by `export-abi.js` — do not edit by hand).

**Routing (Next.js App Router):**
- `/marketplace` — browse listings
- `/item/[id]` — item detail with trading panel (switches on `saleType`)
- `/sell` — list new item
- `/profile` — user orders

**Data flow:** `src/hooks/` contains one hook per contract (`useMarketplace`, `useDirectSale`, `useOpenAuction`, `useBlindAuction`, `useInterest`). Pages consume these hooks; `src/config/contracts.ts` maps chain ID → deployed addresses (populated from `NEXT_PUBLIC_*` env vars).

**`next.config.js`** sets webpack fallbacks for `fs/net/tls` required by wagmi/viem — do not remove these.

## Environment Variables

**`contracts/.env`**
```
SEPOLIA_RPC_URL=
PRIVATE_KEY=
ETHERSCAN_API_KEY=
```

**`frontend/.env.local`**
```
NEXT_PUBLIC_MARKETPLACE_ADDRESS=
NEXT_PUBLIC_INTEREST_CALCULATOR_ADDRESS=
NEXT_PUBLIC_PLATFORM_ADDRESS=
NEXT_PUBLIC_ALCHEMY_ID=
```

## Typical Workflow

1. Edit/deploy contracts → update addresses in `frontend/.env.local`
2. `cd contracts && npm run export-abi` — syncs ABIs to frontend
3. `cd frontend && npm run dev`
